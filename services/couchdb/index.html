CouchDB

database, nosql
Official documentation

Tasks

Name Options Description
start - Start CouchDB if stopped
stop - Stop CouchDB if started
reload - Reload CouchDB, reload the configuration and perform a graceful restart
restart - Restart CouchDB, reload the configuration (but kills existing connection)

Notes

For better user experience you may prefer the use of the reload task instead of restart.

Configuration

Variable Default Description
string admin_pass admin Define the admin password
string admin_user admin Define the admin username
string bind_address 127.0.0.1 The IP address to bind to.
integer enable 1 Define whether CouchDB should start at bood time
boolean insecure false Allow insecure setup (no admin user)
integer max_document_size 4294967296 Maximum document size
integer port 5984 Listening port
string running_user couchdb Define the running user for the CouchDB process (it needs to exist)
string stderr_file /dev/null Where to send the process STDERR
string stdout_file /dev/null Where to send the process STDOUT
