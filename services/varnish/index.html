Varnish Official documentation

Commands

start
Start Varnish if stopped
stop
Stop Varnish if started
reload
Reload Varnish, reload the configuration and perform a graceful restart
restart
Restart Varnish, reload the configuration (but kills existing connection)

Configuration

admin_listen_address string admin_listen_address
Telnet admin interface listen address, usually only bind localhost IP address for the admin interface. Default: 127.0.0.1
admin_listen_port integer admin_listen_port
Telnet admin interface listen port. Default: 6082
listen_address string listen_address
Blank address means all IPv4 and IPv6 interfaces, otherwise specify a host name, an IPv4 dotted quad.
listen_port integer listen_port
Listen port. Default: 6081
max_threads integer max_threads
The maximum number of worker threads to start. Default: 1000
min_threads integer min_threads
The minimum number of worker threads to start. Default: 1
secret_file string secret_file
File containing administration secret. Default: /etc/varnish/secret
storage_file string storage_file
Storage file, only used if storage_type is file. Default: /var/lib/varnish/$INSTANCE/varnish_storage.bin
storage_size string storage_size
Cache size: in bytes, optionally using k / M / G / T suffix. Default: 256M
storage_type string storage_type
Store objects either in memory (malloc), on in a file (file). Default: malloc
thread_timeout integer thread_timeout
Idle timeout for worker threads. Default: 120
ttl integer ttl
Default TTL used when the backend does not specify one. Default: 120
vcl_conf string vcl_conf
Main configuration file. Default: /etc/varnish/default.vcl
