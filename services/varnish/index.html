Varnish

caching, reverse proxy
Official documentation

Varnish is a web application accelerator. You install it in front of your web application and it will speed it up significantly.

Example

services:
  varnish: '*'
configuration:
  varnish:
    listen_port: 80
    listen_address: 0.0.0.0
    storage_type: malloc
    storage_size: 512M

Install Varnish on the node, making it listen on all interfaces (0.0.0.0) and on TCP/80. The cache is set to be held in RAM (Malloc) for a maximum size of 512MB.

Configuration

Variable Default Description
string admin_listen_address 127.0.0.1 Telnet admin interface listen address, usually only bind localhost IP address for the admin interface.
integer admin_listen_port 6082 Telnet admin interface listen port.
string listen_address Blank address means all IPv4 and IPv6 interfaces, otherwise specify a host name, an IPv4 dotted quad.
integer listen_port 6081 Listen port.
integer max_threads 1000 The maximum number of worker threads to start.
integer min_threads 1 The minimum number of worker threads to start.
string secret_file /etc/varnish/secret File containing administration secret.
string storage_file /var/lib/varnish/$INSTANCE/varnish_storage.bin Storage file, only used if storage_type is file.
string storage_size 256M Cache size: in bytes, optionally using k / M / G / T suffix.
string storage_type malloc Store objects either in memory (malloc), on in a file (file).
integer thread_timeout 120 Idle timeout for worker threads.
integer ttl 120 Default TTL used when the backend does not specify one.
string vcl_conf /etc/varnish/default.vcl Main configuration file.
